Deploy on GAS

There are two options for deploying the runtime:

  • Use your web browser to navigate to the deployment page (typically located at http://my.server.com:6394/ws/r/services/DeploymentService/html) and upload the fjs-gbc-<version>-build<build>-customization_<customName>.zip file. This option is only available if you have access to your GAS deployment portal.

  • If the deployment portal is not accessible, you need to have console access to the server and a way to transfer the fjs-gbc-<version>-build<build>-customization_<customName>.zip file to the server. Once the file is on the server, you need to execute the following command from the server console to deploy the GBC:

$ gasadmin gbc --deploy fjs-gbc-<version>-build<build>-customization_<customName>.zip

Once the GBC is deployed, you can render your application using the deployed GBC with the following URL:

http://my.server.com:6394/ua/r/<appName>?gbc=fjs-gbc-<version>-build<build>-customization_<customName>

Or you can set it as the default:

gasadmin gbc --default fjs-gbc-<version>-build<build>-customization_<customName>

The deployed GBC will be used if one uses no query string parameter, as in this example URL:

http://my.server.com:6394/ua/r/<appName>
