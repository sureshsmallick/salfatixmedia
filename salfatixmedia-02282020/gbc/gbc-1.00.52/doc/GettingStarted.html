Getting started

This quick start will prepare you for GBC development and customization.

Prerequisites

You need some tools installed on your system:

  • Node.js — GBC currenlty requires Node 8.x or 10.x
  • git

Follow the tools instruction to install them on your system. Once done, you should have the following tools available from the command line:

$ node -v
v8.9.1
$ npm -v
5.5.1
$ git --version
git version 2.7.4

You need to master these command line tools in order to work on GBC customization.

Next, install grunt using the npm tool. From the console, execute the following command:

$ npm install -g grunt-cli
  • You must have internet access.
  • You may need super user rights on your system.
  • Unless you reinstall node.js or a new version of grunt-cli is required, this command is only needed once.

Your environment is now ready.

Setup your customization project

Get the latest GBC project package (https://4js.com/download/products/) and extract it to a location of your choosing. We will refer to this location where you extract the package as <GBC_PROJECT_DIR> throughout this documentation.

Then in a console, prepare your working copy:

$ cd $GBC_PROJECT_DIR
$ npm install
  • you must have internet access
  • Run this command whenever you get a new version of the project package

And now run the default compilation:

$ grunt

The <GBC_PROJECT_DIR>/dist/customization/sample folder is created and contains the compiled version of GBC with the provided customization sample defined in <GBC_PROJECT_DIR>/customization/sample.

Testing on a GAS

To test your newly built customization, you need first to run GAS by executing the following command:

$ httpdispatch -E res.path.gbc.user=<GBC_PROJECT_DIR>/dist/customization
  • It is very helpful to add the following arguments as well to the httpdispatch command
    • -E res.log.output.type=CONSOLE,DAILYFILE
    • -E res.log.categories_filter=ALL
  • Use a dedicated console as you do not need to reexecute httpdispatch each time you rebuild your GBC customization.

The resource res.path.gbc.user amends the default GBC_LOOKUP_PATH. To run a given application using a given customization, amend the application URL with the query string parameter gbc=<customization_name>. In order to run the Genero demo with the customization you just built above (named sample), open the following URL in your prefered browser:

http://localhost:6394/ua/r/gwc-demo?gbc=sample

Build a project continuously

To make the development cycle quicker, there is a built-in rule that will watch any modification in your customization and instantaneously re-build the customized GBC. You will just have to refresh your browser once compilation is done. To use this, execute the following command:

$ grunt dev
  • Use a dedicated console, as this command won't return and run permanently
  • Depending on your operating system, it is advised to abort and re-run the command if you add new files
